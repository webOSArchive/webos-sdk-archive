MIXIN

moon.MarqueeSupport

From moonstone/source/Marquee.js

Description

The moon.MarqueeSupport mixin should be used with controls that contain multiple marquees whose animation behavior should be synchronized. Calling this.startMarquee() or this.stopMarquee() will start or stop all contained marquees.

The following properties, defined on the base kind to which the mixin is applied, control the marquee behavior:

marqueeOnSpotlight: When true, marquee starts when control is spotlight focused and ends when it is spotlight blurred.

marqueeOnHover: When true, marquee runs while control is hovered over with the mouse. This property is ignored if marqueeOnSpotlight is true.

marqueeOnRender: When true, marquee starts running as soon as control is rendered, and runs continuously.

marqueeSpeed: The speed of the marquee animation, in pixels per second.

marqueeDelay: The delay between spotlight focus/hover and the start of the animation. (This is only used when either marqueeOnSpotlight or marqueeOnHover is true).

marqueeOnRenderDelay: Used when you want the marquee to run on render, after a specified delay.

marqueePause: The duration in milliseconds that the marquee will pause at the end of the animation, before resetting to the beginning.
