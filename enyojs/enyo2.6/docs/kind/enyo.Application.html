enyo.Application

Extends  enyo.Object > enyo.Component > enyo.MultipleDispatchComponent > enyo.Controller > enyo.ViewController
From enyo26/enyo/source/kernel/Application.js

Description

enyo.Application is a type of enyo.ViewController that encapsulates a collection of controllers and a hierarchy of controls. There may be multiple instances of an application at a given time, with unique names and target DOM nodes. Within a given application, a reference to the application is available on all components via the app property.

public $ : Object
Inherited from enyo.Component#$
Default: null

A hash of references to all the components owned by this component. This property is updated whenever a new component is added; the new component may be accessed via its name property. We may also observe changes on properties of components referenced by the $ property.

Component access via the $ hash:

var c = new enyo.Component({
    name: 'me',
    components: [
        {kind: 'Component', name: 'other'}
    ]
});

// We can now access 'other' on the $ hash of 'c', via c.$.other

Observing changes on a component referenced by the $ property:

var c = new enyo.Component({
    name: 'me',
    components: [
        {kind: 'Component', name: 'other'}
    ]
});

c.addObserver('$.other.active', function() {
    // do something to respond to the "active" property of "other" changing
})

c.$.other.set('active', true); // this will trigger the observer to run its callback

public startJob(nom, job, wait, priority)→ this
Inherited from enyo.Component#startJob

Creates a new job tied to this instance of the component. If the component is destroyed, any jobs associated with it will be stopped.

If you start a job with the same name as a pending job, the original job will be stopped; this can be useful for resetting timeouts.

You may supply a priority level (1-10) at which the job should be executed. The default level is 5. Setting the priority lower than 5 (or setting it to the string "low") will defer the job if an animation is in progress, which can help to avoid stuttering.

Parameters

  • nom
    Type: String

    The name of the job to start.

  • job
    Type: function, String

    Either the name of a method or a function to execute as the requested job.

  • wait
    Type: Number

    The number of milliseconds to wait before starting the job.

  • priority
    This parameter is optional
    Default value: 5
    Type: Number

    The priority value to be associated with this job.

Returns

The callee for chaining.
