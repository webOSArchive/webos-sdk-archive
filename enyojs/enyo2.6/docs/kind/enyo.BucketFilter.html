enyo.BucketFilter

Extends  enyo.Object > enyo.Component > enyo.Collection > enyo.Filter
From enyo26/enyo/source/data/BucketFilter.js

Description

A filter designed to maintain multiple filtered sets based on one collection of models. enyo.BucketFilter and all of its filters may be used in conjunction with controls that support usage with enyo.Collection.

Properties

public activeFilter : String
Default: '*'

The currently activated internal filter to proxy. If the name of a filter is given, the enyo.BucketFilter will act as if its content is that of the selected filter. Setting this value to '*' will deselect any filter and will proxy its complete dataset. If this is set to any falsy value, it will be coerced to '*'. During initialization, if any child-filter has the isDefault flag set to true, it will become the active filter instead.

public $ : Object
Inherited from enyo.Component#$
Default: null

A hash of references to all the components owned by this component. This property is updated whenever a new component is added; the new component may be accessed via its name property. We may also observe changes on properties of components referenced by the $ property.

Component access via the $ hash:

var c = new enyo.Component({
    name: 'me',
    components: [
        {kind: 'Component', name: 'other'}
    ]
});

// We can now access 'other' on the $ hash of 'c', via c.$.other

Observing changes on a component referenced by the $ property:

var c = new enyo.Component({
    name: 'me',
    components: [
        {kind: 'Component', name: 'other'}
    ]
});

c.addObserver('$.other.active', function() {
    // do something to respond to the "active" property of "other" changing
})

c.$.other.set('active', true); // this will trigger the observer to run its callback
