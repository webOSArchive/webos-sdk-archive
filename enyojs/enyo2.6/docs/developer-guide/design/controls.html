UI Controls

  • Accordion

    An item that may have child items. Only one additional level is permitted. Multiple accordions may be grouped such that only one accordion in the group may be open at a time.

  • Button

    A control that invokes an action when pressed. Variants include Small Button, Large Button, Icon Button, and Toggle Button.

  • Calendar

    A calendar for selecting dates

  • Checkbox Item

    A control that combines a checkbox with a text label

  • Contextual Popup

    A container that may be invoked by an action, such as pressing a button or hovering over a control

  • Date Picker

    A control in which multiple pickers are combined to allow the user to select a date

  • Divider

    A preformatted control that combines a text label with a horizontal rule. It may be used as a separator between sections in a list, or as a section header inside a panel.

  • Drawer

    A collapsible drawer that sits atop all other UI

  • Expandable Input

    A text input inside an expandable control

  • Expandable Picker

    An expandable control that opens to allow the user to select a value from a list; when it closes, the selected value is returned. This is Moonstone’s recommended replacement for the drop-down menu and list box.

  • Form Checkbox

    A checkbox used in forms; the user activates it to indicate agreement to conditions

  • Header/List Actions

    A predefined area at the top of a panel, containing the header name, an optional subheader, and an area for “list actions”–buttons that enable the user to perform actions on the panel’s contents

  • Image Grid

    A list-based container for displaying images

  • Item/Selectable Item

    A focusable Moonstone-styled control that can display simple text. The “Selectable Item” variant adds support for selection state (selected or unselected).

  • List

    A list may be created by defining a “list item”, which is repeated to create the list. When a list’s contents overflow, a scroller appears.

  • Panel

    A container that emanates from the right side of the screen. Panels are used to represent hierarchy in an application. By default, a Moonstone panel includes a pre-formatted header and a FittableRows layout for the main body content.

  • Popup

    A modal container that may contain controls

  • Progress Bar

    A control that shows the current progress of a process in a horizontal bar

  • Radio Item Group

    A group of radio buttons. The user is allowed to choose one (and only one) item from among a variety of items. Selecting one button will release the previous selection (if there is one). One way this may be used is to allow the selection of a view.

  • Simple Picker

    A basic picker that allows the user to select one item from a list of items

  • Slider

    A control for setting a value from a defined range of values. Used to adjust system settings like contrast and brightness.

  • Text Input

    A region in which text may be displayed or entered. Overflow text will be scrolled.

  • Toggle Button

    A button with “on” and “off” states

  • Toggle Item

    A control used to toggle between two states or actions

  • Tool Tip

    A non-modal popup that contains a text string
